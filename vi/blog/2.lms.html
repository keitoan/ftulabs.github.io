Chạy Moodle CodeRunner trên cụm Docker Swarm

2026-01-25 · FTU Labs · 7 phút đọc

Tụi mình cần một cách để sinh viên có thể viết và nộp code trực tiếp ngay trong các bài kiểm tra trên Moodle, kèm theo đó là kết quả chấm điểm tự động tức thì. Giải pháp của tụi mình là sử dụng CodeRunner kết hợp với hệ thống sandbox Jobe phân tán chạy trên cụm Docker Swarm. Trang web học tập (LMS) của tụi mình hiện đang chạy tại: ftds.online.

Kiến trúc hệ thống

Moodle được cài đặt trên một máy chủ ảo (VPS) của DigitalOcean. Khi sinh viên nộp code, CodeRunner sẽ gửi yêu cầu chấm bài tới địa chỉ jobe.ftds.online. Địa chỉ này thực chất là một đường truyền ngược (Reverse Tunnel) qua frp dẫn thẳng vào cụm Docker Swarm đặt tại phòng lab của tụi mình. HAProxy sẽ chịu trách nhiệm cân bằng tải giữa các bản sao Jobe — mỗi máy trong cụm Swarm sẽ chạy một bản sao Jobe. 

Sinh viên -> ftds.online (DigitalOcean VPS)
               ├── ftds.online       -> Moodle / Apache
               └── jobe.ftds.online  -> quy trình frp tunnel
                                         ↓
                                     HAProxy LB
                                         ↓
                               Các bản sao Jobe (1 bản mỗi máy)
  • Moodle + Apache: Chạy trên máy chủ ảo (VPS), đóng vai trò là giao diện chính của trang học tập.
  • Jobe (jobeinabox): Môi trường sandbox để thực thi code, chạy đồng loạt trên tất cả các máy trong cụm Swarm.
  • HAProxy: Bộ cân bằng tải kèm theo các bước kiểm tra trạng thái (Health Check) (inter 10s, fall 5, rise 2).
  • frp tunnel: Một dịch vụ trong Docker Swarm giúp đưa bộ cân bằng tải ra ngoài internet thông qua jobe.ftds.online.

Thiết lập cụm Swarm

Tất cả các máy phải nằm trong cùng một mạng nội bộ. Tụi mình thiết lập các router ở chế độ Access Point (AP) để mọi thiết bị có cùng dải IP, sau đó cài đặt từng máy bằng một tập lệnh tự động: 

# Trên máy chính (leader) — khởi tạo Docker Swarm
docker swarm init --advertise-addr 10.0.190.70:2377

# Trên các máy con (worker) — điền token và IP máy chính vào file .env, sau đó chạy:
sudo ./swarm

Tập lệnh swarm sẽ tự động cài Docker (hỗ trợ Ubuntu/Arch), cấp quyền người dùng và thực hiện gia nhập vào cụm Swarm một cách tự động.

Triển khai hạ tầng

Khi cụm Swarm đã sẵn sàng, tụi mình triển khai 3 dịch vụ chính: các bản sao Jobe, HAProxy để cân bằng tải và đường truyền frp để kết nối cụm máy tính với Moodle VPS. 

# Nạp các file cấu hình
docker config create haproxy_cfg ./haproxy.cfg
docker config create frpc_cfg ./frpc.cfg

# Triển khai bộ CodeRunner
docker stack deploy -c stack.yml coderunner

# Triển khai Portainer để theo dõi
docker stack deploy -c portainer-agent-stack.yml portainer

# Kiểm tra trạng thái
docker stack ps coderunner

Portainer cung cấp một giao diện web tại cổng 9000 giúp tụi mình theo dõi sức khỏe của các máy, trạng thái các container và biểu đồ trực quan của toàn bộ cụm Swarm.

Đường truyền frp (frp Tunnel)

Vì cụm máy tính nằm trong mạng nội bộ (phía sau NAT), tụi mình dùng frp để tạo một đường truyền ngược từ VPS về phòng lab. VPS sẽ chạy frps tại cổng 7000, còn máy chính ở lab chạy frpc dưới dạng một dịch vụ Docker: 

# frpc.toml (chạy trên máy leader ở lab)
serverAddr = "ftds.online"
serverPort = 7000
auth.token = "..."

[[proxies]]
name = "jobe_tunnel"
type = "http"
localIP = "lb"           # trỏ tới mạng nội bộ của Docker
localPort = 80
customDomains = ["jobe.ftds.online"]

Phía trên VPS, Apache sẽ nhận các yêu cầu tới jobe.ftds.online và đẩy ngược về cổng 8080 của frp: 

# /etc/apache2/sites-available/jobe.conf
<VirtualHost *:80>
    ServerName jobe.ftds.online
    ProxyPreserveHost On
    ProxyPass / http://127.0.0.1:8080/
    ProxyPassReverse / http://127.0.0.1:8080/
</VirtualHost>

Hỗ trợ SQL với sqlglot

Tụi mình đã build một image Jobe riêng tích hợp sẵn thư viện sqlglot để chuyển đổi cú pháp SQL (SQL transpilation). Sinh viên có thể viết code SQL theo cú pháp của MySQL hoặc T-SQL, và sandbox sẽ tự động chuyển đổi nó sang SQLite trước khi thực thi. Điều này giúp tụi mình sử dụng được bộ dữ liệu mẫu AdventureWorksLT (được chuyển từ file .bak của SQL Server sang SQLite) mà không cần phải cài đặt một máy chủ database cồng kềnh. 

# Build image Jobe tùy chỉnh kèm sqlglot
docker build -t ${REGISTRY}/jobe-sqlglot:latest ./jobe
docker push ${REGISTRY}/jobe-sqlglot:latest

# Triển khai lại
docker stack deploy -c docker-compose.yml coderunner

Mẫu bài tập trên CodeRunner sử dụng lệnh sqlglot.transpile(sql, read="tsql", write="sqlite") để xử lý các cú pháp T-SQL như TOP 5, ISNULL() và các định danh trong dấu ngoặc vuông.

Quản lý khóa học

Tụi mình có viết sẵn một tập lệnh shell để nhân bản (clone) khóa học cho mỗi kỳ học mới — giúp sao chép toàn bộ các hoạt động, bài kiểm tra, ngân hàng câu hỏi và tài liệu trong khi vẫn giữ sạch dữ liệu sinh viên cũ: 

# Nhân bản khóa học cho kỳ học mới
./scripts/clone-course.sh 8 "TIN313(2627-1)GD1.01" \
  "TIN313(2627-1)GD1.01 - CO SO DU LIEU - Minh" "2026-09-01"

├── swarm.ps1               # Bản dành cho Windows
├── allow-ports             # Cấu hình tường lửa
├── stack.yml               # Docker stack (Jobe + HAProxy + frpc)
└── haproxy.cfg             # Cấu hình bộ cân bằng tải

ftulabs/moodle-coderunner   # Tích hợp Moodle
├── docker-compose.yml      # Swarm stack
├── jobe/
│   ├── Dockerfile          # Image Jobe tùy biến (base + sqlglot)
│   └── haproxy.cfg         # Cấu hình HAProxy
├── scripts/
│   ├── clone-course.sh     # Script nhân bản khóa học
│   └── clone-course.php    # Sao lưu/khôi phục trên Moodle
└── adventureworks/
    ├── AdventureWorksLT2019.db
    └── sql_quizzes.xml     # Ngân hàng câu hỏi (Moodle XML)
