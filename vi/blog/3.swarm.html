Dựng cụm máy tính 40 node (Cluster) cho LLM bằng Docker Swarm

2026-02-25 · FTU Labs · 8 phút đọc

Tụi mình có sẵn 88 chiếc laptop dùng chip i7-13700H, RAM 16GB và card đồ họa tích hợp Intel UHD 770 trong phòng lab. Thay vì để không, tụi mình muốn biến chúng thành một cụm máy tính tự vận hành để suy luận (inference) LLM chạy 24/7 — không dùng cloud, không tốn tiền mua API key, không mất đồng xu lẻ nào. Và đây là cách tụi mình đã thực hiện.

Cụm máy tính này hiện đang chạy tại swarm.ftds.online. Toàn bộ mã nguồn có tại: ftulabs/cluster.

Kiến trúc hệ thống

Thiết kế khá đơn giản: một node chính (leader) sẽ chạy bộ điều phối (Orchestrator), bộ cân bằng tải (Load Balancer) và hệ thống theo dõi (Monitoring). Các node còn lại đóng vai trò công việc (worker), mỗi máy sẽ chạy một mô hình LLM đã được lượng tử hóa (Quantized) thông qua công cụ llama.cpp. Ngoài ra, tụi mình có một chiếc Nvidia Jetson Xavier để chạy model 35B MoE lớn hơn bằng CUDA nhằm cho ra các câu trả lời chất lượng cao hơn.

Công nghệ sử dụng

  • Docker Swarm: điều phối (Orchestration) trên toàn bộ hơn 40 node.
  • llama.cpp: thực hiện suy luận (Inference) mô hình GGUF, tạo API tương thích với OpenAI trên mỗi máy worker.
  • nginx: làm proxy với thuật toán Round-Robin có trọng số (Weighted Round-Robin) — node Xavier được ưu tiên gấp 4 lần so với các máy i7.
  • CrewAI: bộ điều phối đa tác vụ (Multi-agent Orchestrator) chạy trên node chính.
  • Redis: lưu trữ trạng thái chung cho cả hệ thống.
  • Prometheus + Grafana: theo dõi các chỉ số và hiển thị biểu đồ.
  • Open WebUI: giao diện chat tại cổng 3001.
  • Ansible: tự động hóa quá trình thiết lập (bootstrap) cho tất cả các node.

Quá trình triển khai

Tụi mình tự động hóa toàn bộ quá trình cài đặt bằng Ansible. Chỉ với một tập lệnh (Playbook), hệ thống sẽ tự động cài Docker, nhận diện card đồ họa GPU, khởi tạo cụm Swarm, dán nhãn các node, phân phối mô hình (~2.6GB/máy), build image và triển khai toàn bộ hạ tầng. 

# Cấu hình danh sách máy
# ansible/inventory.ini
[leader]
10.0.190.65 ansible_user=minh

[workers]
10.0.190.101 ansible_user=worker
10.0.190.102 ansible_user=worker
# ... danh sách lên tới 40 node

# Chạy quá trình thiết lập
ansible-playbook -i ansible/inventory.ini ansible/bootstrap.yml

# Kiểm tra kết quả
curl http://10.0.190.65:8888/v1/models
curl http://10.0.190.65:3001  # Giao diện Web UI

Nếu bạn muốn test nhanh trên máy cá nhân, bạn có thể chạy một node duy nhất bằng Docker Compose: 

./scripts/download-model.sh
docker compose -f docker-compose.local.yml up -d
./scripts/test-agent.sh

Cơ chế định tuyến và Cân bằng tải

nginx đóng vai trò định tuyến các yêu cầu (request) bằng thuật toán Round-Robin có trọng số. Node Xavier (chạy model 35B MoE, tốc độ khoảng 11.4 token/giây bằng CUDA) được gán weight=4, giúp nó xử lý khoảng 80% lưu lượng truy cập khi rảnh. 40 node worker (chạy Qwen3.5-4B, tốc độ ~2 token/máy/giây, mỗi máy có 4 khe xử lý song song) sẽ đảm nhận phần việc còn lại khi hệ thống bị quá tải.

Khi có hơn 4 người dùng cùng lúc, các yêu cầu sẽ tự động được đẩy vào nhóm máy worker. API của hệ thống hoàn toàn tương thích với chuẩn OpenAI — nghĩa là bất kỳ công cụ nào hỗ trợ OpenAI API đều có thể dùng ngay lập tức mà không cần API key trong mạng nội bộ. 

# Chat qua API (dùng được với bất kỳ client nào hỗ trợ OpenAI)
curl http://10.0.190.65:8888/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-3.5-turbo",
    "messages": [{"role":"user","content":"Xin chào!"}],
    "max_tokens": 500
  }'

Điều phối đa tác vụ (Multi-Agent Orchestration)

Dịch vụ CrewAI master trên node leader sẽ điều phối các quy trình làm việc đa tác vụ. Bạn chỉ cần gửi một mục tiêu (Goal), bộ điều phối sẽ tự động chia nhỏ thành các nhiệm vụ con cho các worker, theo dõi tiến độ qua Redis và trả về kết quả cuối cùng sau khi đã tổng hợp xong. 

# Gửi một mục tiêu
curl -X POST http://localhost:9091/goal \
  -H "Content-Type: application/json" \
  -d '{"goal": "Phân tích 5 ngôn ngữ lập trình phổ biến nhất hiện nay"}'

# Kiểm tra các máy worker
curl http://localhost:9091/workers

# Xem hiệu năng hệ thống
curl http://localhost:9091/performance

# Kích hoạt quá trình căn chỉnh máy chủ
curl -X POST http://localhost:9091/calibrate

Tổng quan các dịch vụ

Dịch vụ Cổng Mô tả
Worker (LLM) 8080 Server llama.cpp, API chuẩn OpenAI
Master (Điều phối) 9091 Bộ điều phối CrewAI, nhận yêu cầu tại /goal
Open WebUI 3001 Giao diện chat chính
nginx proxy 8888 Proxy cân bằng LLM theo trọng số
Redis 6379 Lưu trữ trạng thái hệ thống
Prometheus 9090 Theo dõi chỉ số
Grafana 3000 Bảng biểu (Dashboard)

Những gì tụi mình đã học được

  • Các mô hình lượng tử hóa 4B chạy trên CPU tuy chậm (~2 token/giây) nhưng hoàn toàn có thể dùng được nếu bạn có 40 máy chạy song song đằng sau một bộ cân bằng tải.
  • Docker Swarm thực sự bị đánh giá thấp — nó đơn giản, ổn định và đủ tốt cho quy mô của tụi mình. Không cần phải dùng tới Kubernetes làm gì cho phức tạp.
  • Điểm nghẽn (bottleneck) lớn nhất lại nằm ở phần thiết lập mạng: làm sao để kết nối hơn 40 máy vào cùng một mạng con (subnet) và duy trì kết nối ổn định.
  • Ansible chính là chìa khóa giúp rút ngắn thời gian từ "mất một tuần để cài đặt" xuống còn "chỉ mất 20 phút".
