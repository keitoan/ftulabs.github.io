Survival Guide cho sinh viên CS tại FTU

2026-03-28 · FTU Labs · 20 phút đọc

Bài viết này tổng hợp những kỹ năng quan trọng mà mọi sinh viên Khoa học máy tính (CS) tại FTU nên trang bị (từ việc cài đặt môi trường lập trình (IDE) cho đến cách hoàn thiện một dự án thực thụ). Mỗi phần sẽ tập trung vào tính thực hành, ngắn gọn và có kèm các nguồn tham khảo chất lượng. Bạn hãy lưu lại trang này để xem lại khi cần nhé.

1. Cài đặt môi trường lập trình (IDE)

Tụi mình khuyên dùng VS Code. Đây là trình soạn thảo mã nguồn miễn phí, cực kỳ linh hoạt và chạy tốt trên mọi hệ điều hành. 

# Cài đặt VS Code (Ubuntu/Debian)
sudo apt update && sudo apt install code

# macOS (qua Homebrew)
brew install --cask visual-studio-code

Các tiện ích mở rộng (Extension) cần thiết (bạn hãy cài trong mục Extensions, phím tắt Ctrl+Shift+X):

  • Python (IntelliSense, kiểm lỗi (linting), gỡ lỗi (debugging))
  • Pylance (kiểm tra kiểu dữ liệu nhanh)
  • GitLens (xem lịch sử chỉnh sửa code trực tiếp trên từng dòng)
  • Prettier (tự động căn chỉnh định dạng code HTML/CSS/JS)
  • Remote - SSH (chỉnh sửa file trực tiếp trên các server từ xa)
  • Live Server (tự động tải lại trang khi làm các dự án web HTML/CSS)

Một vài phím tắt hữu ích:

Ctrl+P           # Mở file nhanh
Ctrl+Shift+P     # Mở thanh lệnh (Command palette)
Ctrl+`            # Mở/tắt terminal
Ctrl+D            # Chọn các từ trùng nhau
Alt+Up/Down       # Di chuyển dòng lên/xuống
Ctrl+Shift+K      # Xóa nhanh một dòng

2. Môi trường ảo (Virtual Environment)

Đừng bao giờ cài thư viện (package) trực tiếp vào hệ thống (global). Hãy dùng môi trường ảo (Virtual Environment) để quản lý riêng các thư viện (Dependency) cho từng dự án.

Cách A: venv (có sẵn trong Python)

# Tạo môi trường ảo
python -m venv .venv

# Kích hoạt
source .venv/bin/activate    # Linux/macOS
.venv\Scripts\activate       # Windows

# Cài đặt thư viện
pip install numpy pandas torch

# Lưu danh sách thư viện hiện có
pip freeze > requirements.txt

# Cài đặt lại từ danh sách có sẵn
pip install -r requirements.txt

# Thoát khỏi môi trường ảo
deactivate

Cách B: conda (dành cho Data Science / CUDA)

# Cài đặt Miniconda (bản nhẹ)
# https://docs.conda.io/en/latest/miniconda.html

# Tạo môi trường mới
conda create -n myproject python=3.11 -y
conda activate myproject

# Cài đặt qua conda-forge
conda install -c conda-forge numpy pandas

# Xuất file cấu hình
conda env export > environment.yml

# Tạo lại môi trường từ file
conda env create -f environment.yml

Cách C: uv (nhanh và hiện đại)

# Cài đặt uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# Tạo dự án kèm môi trường ảo
uv init myproject && cd myproject
uv venv

# Thêm thư viện (tự động xử lý xung đột)
uv add numpy pandas torch

# Đồng bộ (cài đặt mọi thứ từ pyproject.toml)
uv sync
Lời khuyên: dùng venv cho các dự án đơn giản, conda khi cần dùng CUDA hoặc các thư viện hệ thống phức tạp, và dùng uv nếu bạn muốn tốc độ nhanh nhất.

Nguồn tham khảo:

  • Tài liệu về Python venv
  • Tài liệu về uv
  • Fireship: Giải thích về Python Virtual Envs

3. GitHub, Clean Code và quy trình đóng góp

3.1 Tạo mã định danh cá nhân (PAT) của GitHub

# Vào: GitHub (Settings) -> Developer settings -> Personal access tokens -> Fine-grained tokens
# Chọn quyền: Contents (read/write), Pull requests (read/write)
# Copy mã token này

# Cấu hình git (làm một lần duy nhất)
git config --global user.name "Tên của bạn"
git config --global user.email "email@example.com"

# Clone repo bằng token (HTTPS)
git clone https://<TOKEN>@github.com/ftulabs/your-repo.git

# Hoặc dùng SSH (khuyên dùng) — tạo khóa mới:
ssh-keygen -t ed25519 -C "email@example.com"
cat ~/.ssh/id_ed25519.pub
# Add khóa public tại: GitHub (Settings) -> SSH keys

3.2 Quy trình Git cơ bản

# Cập nhật bản mới nhất từ nhánh main
git checkout main && git pull

# Tạo nhánh tính năng (branch)
git checkout -b feature/add-login-page

# Làm việc, lưu thay đổi
git add src/login.py tests/test_login.py
git commit -m "Thêm trang login và xử lý form validation"

# Đẩy mã lên GitHub và tạo Pull Request (PR)
git push -u origin feature/add-login-page
gh pr create --title "Add login page" --body "Xử lý xong phần giao diện đăng nhập"

3.3 Cách viết Code sạch (Clean Code)

# CHƯA TỐT: tên hàm mơ hồ, dùng hằng số trần, thiếu kiểu dữ liệu
def calc(x):
    return x * 1.1

# TỐT: tên rõ ràng, có gợi ý kiểu dữ liệu (Type Hint) và docstring
def calculate_price_with_tax(base_price: float, tax_rate: float = 0.1) -> float:
    """Tính toán tổng giá tiền sau thuế."""
    return base_price * (1 + tax_rate)

Quy tắc viết comment:

  • Hãy giải thích tại sao bạn viết như vậy, thay vì viết cái gì. Bản thân code đã nói lên nó đang làm gì rồi, comment dùng để giải thích ý đồ của bạn.
  • Dùng docstrings cho các hàm công khai. Đừng lạm dụng comment trên từng dòng code.
  • Xóa bỏ những đoạn code thừa bị comment lại, lịch sử git sẽ lưu trữ chúng giúp bạn.

3.4 Quy trình đóng góp (cho các dự án tại FTU Labs)

  1. Fork lại repo (hoặc clone trực tiếp nếu bạn có quyền chỉnh sửa)
  2. Tạo nhánh (branch) từ main với tên gợi nhớ
  3. Viết code và bộ kiểm thử (test), hãy nhớ tuân thủ phong cách code có sẵn
  4. Viết commit với thông điệp rõ ràng (search về feat:, fix:, docs:)
  5. Đẩy mã (Push) và mở một Pull Request (PR)
  6. Phản hồi các góp ý (Review) từ mọi người rồi thực hiện merge
# Ví dụ về Conventional Commits
git commit -m "feat: thêm api đăng nhập người dùng"
git commit -m "fix: xử lý lỗi tranh chấp dữ liệu (race condition) trong hàng đợi"
git commit -m "docs: cập nhật tài liệu API cho các route v2"
git commit -m "refactor: tách logic xử lý dữ liệu vào file utils"

Nguồn tham khảo:

  • GitHub Quickstart (tài liệu chính thức)
  • Quy chuẩn Commits thông dụng
  • freeCodeCamp: Git và GitHub dành cho người mới bắt đầu (1 tiếng)
  • Fireship: Git trong 100 giây

4. LaTeX, Overleaf và arXiv

4.1 Cơ bản về LaTeX

LaTeX là tiêu chuẩn để viết các bài báo khoa học. Công cụ Overleaf giúp bạn soạn thảo trực tiếp trên web mà không cần cài đặt gì phức tạp. 

\documentclass{article}
\usepackage[utf8]{inputenc}
\usepackage{amsmath, amssymb, graphicx, hyperref}

\title{Tiêu đề bài báo của bạn}
\author{Tên của bạn \\ FTU Labs}
\date{\today}

\begin{document}
\maketitle
\begin{abstract}
  Tóm tắt ngắn gọn nội dung công việc của bạn trong khoảng 150-250 từ.
\end{abstract}

\section{Mở đầu}
Nội dung phần mở đầu ở đây. Trích dẫn bằng lệnh~\cite{vaswani2017attention}.

\section{Phương pháp}
Một công thức toán học:
\begin{equation}
  \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
\end{equation}

Cách chèn hình ảnh:
\begin{figure}[h]
  \centering
  \includegraphics[width=0.8\textwidth]{figures/architecture.png}
  \caption{Tổng quan kiến trúc mô hình.}
  \label{fig:arch}
\end{figure}

\bibliographystyle{plain}
\bibliography{references}
\end{document}

4.2 Một vài mẹo dùng Overleaf

  • Dùng tính năng Overleaf Git integration để clone dự án về máy, sửa code tại local rồi push ngược lại lên web.
  • Sử dụng các mẫu bài báo (Template) có sẵn từ các hội nghị lớn như NeurIPS, ICML, ACL, AAAI.
  • Dùng lệnh \input{sections/method.tex} để chia bài báo thành nhiều file nhỏ cho dễ quản lý.
  • Quản lý trích dẫn bằng file .bib (lấy file này từ các trang như Google Scholar hoặc Semantic Scholar).

4.3 Nộp bản thảo lên arXiv

# Chuẩn bị thư mục nộp bài
mkdir arxiv_submission && cd arxiv_submission

# Copy các file nguồn (đừng copy file PDF nhé)
cp main.tex references.bib figures/ -r .

# Gộp các file nhỏ lại nếu cần thiết
latexpand main.tex > merged.tex

# Tạo file nén tarball
tar -czf submission.tar.gz *.tex *.bib *.bst figures/

# Tải lên tại https://arxiv.org/submit
# Chọn danh mục phù hợp (cs.CL, cs.LG, cs.CV, v.v.)
# arXiv sẽ tự biên dịch LaTeX trên server của họ

Nguồn tham khảo:

  • Overleaf: Học LaTeX trong 30 phút
  • Hướng dẫn LaTeX cho người mới bắt đầu (Michelle Krummel)
  • Hướng dẫn nộp bài lên arXiv
  • Mẹo hay khi viết bài báo (GitHub)

6.3 Các máy chủ giao thức MCP (Model Context Protocol)

MCP cho phép các trợ lý AI kết nối với các công cụ bên ngoài (cơ sở dữ liệu, API, hệ thống file, trình duyệt). Bạn có thể hiểu nó như các "plugin" cho AI. 

// .claude/settings.json — ví dụ cấu hình MCP
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-filesystem", "/duong/dan/thu/muc"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-github"],
      "env": { "GITHUB_TOKEN": "ghp_xxx" }
    }
  }
}

7. Học thiết kế hệ thống (System Design)

Thiết kế hệ thống (System Design) là kỹ năng xây dựng kiến trúc phần mềm sao cho có thể mở rộng (scale) được. Ngay cả khi không đi phỏng vấn, việc hiểu các mô hình này cũng giúp bạn trở thành một kỹ sư giỏi hơn.

7.1 Các kiến thức cốt lõi (nên học theo thứ tự)

  1. Mô hình Client-Server (HTTP, các giao thức REST API)
  2. Cơ sở dữ liệu (Database) (SQL đối đầu NoSQL, lập chỉ mục (index), sao chép (replication))
  3. Bộ nhớ đệm (Caching) (Redis, CDN, quy tắc xóa cache)
  4. Cân bằng tải (Load balancing) (các thuật toán như round-robin, consistent hashing)
  5. Hàng đợi thông điệp (Message queue) (Kafka, RabbitMQ, các mô hình pub/sub)
  6. Microservices so với Monolith (ưu nhược điểm và thời điểm cần chuyển đổi)
  7. Định lý CAP (tính nhất quán (Consistency), tính sẵn dùng (Availability), tính chống chịu phân mảnh (Partition tolerance))

7.2 Quy trình thực hành

Khi giải một bài toán System Design, hãy làm theo các bước sau:

1. YÊU CẦU          - Hệ thống dùng để làm gì? (tính năng và hiệu năng mong muốn)
2. ƯỚC TÍNH         - Bao nhiêu dữ liệu? Bao nhiêu người dùng? Số truy vấn mỗi giây (QPS)?
3. THIẾT KẾ TỔNG QUAN - Vẽ sơ đồ các thành phần (API, Service, DB, Cache)
4. CHI TIẾT         - Đi sâu vào phần khó nhất (mô hình dữ liệu, thuật toán)
5. ĐIỂM NGHẼN        - Cái gì sẽ bị hỏng khi hệ thống quá tải? Cách xử lý là gì?

Ví dụ: thiết kế một hệ thống thu gọn link (URL shortener)
- Tính năng: rút gọn link, chuyển hướng, thống kê lượt click
- Quy mô: 100 triệu link, 1000 lượt ghi mỗi giây, 10.000 lượt đọc mỗi giây
- Thành phần: API Gateway -> Shortening Service -> DB (Key-value)
- Quyết định then chốt: dùng mã hóa base62 hay băm link, ưu tiên đọc nhiều nên cần thêm lớp Cache

Nguồn tham khảo:

  • System-design-primer (GitHub cực kỳ nổi tiếng)
  • ByteByteGo: System Design 101
  • Kênh YouTube của ByteByteGo
  • Hussein Nasser: Khóa học System Design đầy đủ
  • Sách: Designing Data-Intensive Applications của Martin Kleppmann
  • Sách: System Design Interview của Alex Xu

8. Dự án từ đầu đến cuối: Từ ý tưởng đến CI/CD

Đây là lúc mọi thứ kết nối lại với nhau. Đây là vòng đời đầy đủ của một dự án thực tế.

Giai đoạn 1: Ý tưởng và Lên kế hoạch

Xác định bài toán:
- Bạn đang giải quyết vấn đề gì?
- Ai là người dùng cuối?
- Hiện đã có những giải pháp gì rồi? (tìm hiểu tài liệu hoặc đối thủ)

Viết một bản tóm tắt (One-pager):
- Mô tả vấn đề (ngắn gọn trong 2-3 câu)
- Giải pháp đề xuất
- Các tiêu chí đo lường sự thành công
- Lựa chọn công nghệ sử dụng và lý do tại sao

Tạo repo trên GitHub với cấu trúc chuẩn:
my-project/
├── README.md            # Mô tả dự án, cách chạy
├── pyproject.toml       # Thư viện và metadata
├── src/
│   └── my_project/
│       ├── __init__.py
│       ├── model.py
│       └── api.py
├── tests/
│   └── test_model.py
├── docs/
├── .github/

Giai đoạn 3: Triển khai

Dùng các nhánh (branch) riêng cho từng tính năng
git checkout -b feat/model-inference

Viết các bộ kiểm thử (test) TRƯỚC (hoặc song song)
# tests/test_model.py
def test_predict_returns_label():
    result = predict("xin chào")
    assert result["label"] in ["tích cực", "tiêu cực", "trung tính"]
    assert 0 <= result["confidence"] <= 1

Chạy test tại local trước khi push code
pytest tests/ -v

Giai đoạn 4: Quy trình CI/CD

# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -e ".[dev]"
      - run: ruff check src/
      - run: pytest tests/ -v --tb=short

  deploy:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy lên server
        run: |
          echo "Thực hiện bước deploy tại đây"
          # Ví dụ: build docker, đẩy lên registry, deploy lên cloud

Quy trình hoàn chỉnh

Ý tưởng -> Tóm tắt -> Thiết kế hệ thống -> Cài đặt Repo
  -> Tạo nhánh + Viết code + Viết test -> PR + Review -> CI pass
  -> Merge vào main -> Tự động Deploy -> Viết bài báo khoa học
  -> Làm trang web dự án -> Đưa lên arXiv -> Chia sẻ với cộng đồng
Khoảng cách giữa một dự án của sinh viên và một dự án chuyên nghiệp không nằm ở tài năng mà nằm ở quy trình. Nếu bạn tuân thủ các bước này một cách nhất quán, bạn sẽ tạo ra được những thứ có giá trị bền lâu.
